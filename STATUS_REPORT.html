Executive Summary

A snapshot of everything that was shipped across both repositories during this work cycle.

The Order Management System is now fully operational from end to end. The backend REST API, owned by Rimjhim under the rimjhimkrishna account, has been deployed to production on Railway together with a managed PostgreSQL database and an Apache Kafka event-streaming cluster. The React storefront, owned by Sangam under the SnakeEye-sudo account, has been connected to the live API, redeployed on Vercel, seeded with a working product catalogue, and given a refreshed, more vibrant visual identity. A complete registration → login → catalogue → order journey has been verified directly against the live environment.

Backend API rimjhimkrishna Live

Spring Boot 3 · Java 17 · PostgreSQL · Apache Kafka · Docker

What was delivered

  • Provisioned a new Railway project (perpetual-grace) and connected it to the backend repository.
  • Modernised the multi-stage Dockerfile to use current, supported base images so the production build succeeds.
  • Added a managed PostgreSQL database and wired the application to it through Railway service references.
  • Stood up an Apache Kafka broker in KRaft mode (no ZooKeeper) for the order event stream, including topic provisioning and a Dead Letter Topic for resilience.
  • Configured all runtime environment variables: database URL/credentials, Kafka bootstrap server, JWT secret and token expiry.
  • Hardened the CORS policy so the deployed storefront can call the API securely from its own origin.
  • Generated a public production domain and confirmed the Actuator health endpoint reports UP.
  • Seeded the catalogue with eight sample products through the authenticated admin API.
  • Updated the project README with a dedicated Live Demo section linking to the running services.
